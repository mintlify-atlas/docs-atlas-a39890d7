---
title: 'dv-cloak'
description: 'Hide uncompiled template content during initialization'
---

The `dv-cloak` directive is used to hide uncompiled template content while your DativeJS application is initializing. It remains on the element until the component finishes compiling and rendering.

## Syntax

```html
<element dv-cloak></element>
```

The directive takes no value. If you provide a value, DativeJS will warn you that the directive doesn't accept values.

## Purpose

When a DativeJS application loads, there can be a brief moment where users see uncompiled template syntax (like `{{ variable }}`) before JavaScript executes. The `dv-cloak` directive solves this by:

1. Staying on the element while the component compiles
2. Being automatically removed once compilation is complete
3. Working with CSS to hide uncompiled content

## Usage with CSS

To use `dv-cloak` effectively, you need to add a corresponding CSS rule:

```css
[dv-cloak] {
  display: none;
}
```

This CSS rule hides any element with the `dv-cloak` attribute. Once DativeJS finishes compiling, it removes the attribute, making the element visible.

## Complete Example

### HTML

```html
<!DOCTYPE html>
<html>
<head>
  <style>
    [dv-cloak] {
      display: none;
    }
  </style>
</head>
<body>
  <div id="app"></div>
  <script src="dative.js"></script>
  <script src="app.js"></script>
</body>
</html>
```

### JavaScript

```javascript
new Dative({
  target: '#app',
  data() {
    return {
      message: 'Hello, World!',
      user: {
        name: 'John Doe',
        email: 'john@example.com'
      }
    }
  },
  template: `
    <div dv-cloak>
      <h1>{{ message }}</h1>
      <p>Welcome, {{ user.name }}!</p>
      <p>Email: {{ user.email }}</p>
    </div>
  `
})
```

## Usage Examples

### Basic Usage

```html
<div dv-cloak>
  <h1>{{ title }}</h1>
  <p>{{ description }}</p>
</div>
```

Users won't see `{{ title }}` or `{{ description }}` during loading - they'll see nothing until the component is ready.

### With Loading Indicator

```html
<style>
  [dv-cloak] {
    display: none;
  }
  
  .loading {
    display: block;
  }
  
  .loading[dv-cloak] {
    display: none;
  }
</style>

<div class="loading">Loading...</div>
<div dv-cloak>
  <h1>{{ content }}</h1>
</div>
```

### Entire Application

```html
<style>
  [dv-cloak] {
    display: none;
  }
</style>

<div id="app" dv-cloak>
  <!-- Your entire app template -->
  <header>{{ headerTitle }}</header>
  <main>{{ mainContent }}</main>
  <footer>{{ footerText }}</footer>
</div>
```

### With Fade-In Effect

```html
<style>
  [dv-cloak] {
    opacity: 0;
    transition: opacity 0.3s;
  }
  
  /* This class can be added after dv-cloak is removed */
  .visible {
    opacity: 1;
  }
</style>

<div dv-cloak>
  <h1>{{ title }}</h1>
</div>
```

### Selective Hiding

```html
<style>
  [dv-cloak] {
    display: none;
  }
</style>

<div>
  <!-- This shows immediately -->
  <header>
    <img src="logo.png" alt="Logo">
  </header>
  
  <!-- This is hidden until compiled -->
  <main dv-cloak>
    <h1>{{ dynamicTitle }}</h1>
    <p>{{ dynamicContent }}</p>
  </main>
</div>
```

## Implementation Details

From the source code (`index.js:1047-1058`):

```javascript
else if (name === prefix("cloak")) {
  if (el.getAttribute(name) !== "") {
    warn(
      "`dv-cloak` doesn't accepts any value.\nShould be\n<" +
        el.tagName.toLowerCase() +
        " dv-cloak ...></" +
        el.tagName.toLowerCase() +
        ">"
    );
  }
  queueMicrotask(() => el.removeAttribute(name));
}
```

Key points:
- The directive checks if a value was provided and warns if so
- Uses `queueMicrotask()` to remove the attribute after the current task
- This ensures the attribute is removed once rendering completes

## Best Practices

### 1. Always Include CSS

```css
/* Required for dv-cloak to work */
[dv-cloak] {
  display: none;
}
```

Without this CSS rule, the directive won't hide anything.

### 2. Use on Root Elements

```html
<!-- Good: Applied to root container -->
<div dv-cloak>
  <h1>{{ title }}</h1>
  <p>{{ content }}</p>
</div>

<!-- Less efficient: Multiple dv-cloak attributes -->
<div>
  <h1 dv-cloak>{{ title }}</h1>
  <p dv-cloak>{{ content }}</p>
</div>
```

### 3. Consider Loading States

```html
<style>
  [dv-cloak] { display: none; }
  .app-loading { display: flex; justify-content: center; }
</style>

<div class="app-loading">Loading application...</div>
<div dv-cloak>
  <!-- App content -->
</div>
```

### 4. Don't Provide Values

```html
<!-- Correct -->
<div dv-cloak></div>

<!-- Wrong: Will trigger a warning -->
<div dv-cloak="true"></div>
<div dv-cloak="someValue"></div>
```

## Common Patterns

### Loading Spinner

```html
<style>
  [dv-cloak] { display: none; }
  .spinner {
    border: 4px solid #f3f3f3;
    border-top: 4px solid #3498db;
    border-radius: 50%;
    width: 40px;
    height: 40px;
    animation: spin 1s linear infinite;
  }
  @keyframes spin {
    0% { transform: rotate(0deg); }
    100% { transform: rotate(360deg); }
  }
</style>

<div class="spinner"></div>
<div dv-cloak>
  <h1>{{ appTitle }}</h1>
  <!-- App content -->
</div>
```

### Skeleton Screen

```html
<style>
  [dv-cloak] { display: none; }
  .skeleton {
    background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
    background-size: 200% 100%;
    animation: loading 1.5s infinite;
  }
  @keyframes loading {
    0% { background-position: 200% 0; }
    100% { background-position: -200% 0; }
  }
</style>

<!-- Skeleton placeholder -->
<div class="skeleton" style="width: 200px; height: 20px;"></div>

<!-- Actual content -->
<div dv-cloak>
  <h1>{{ actualContent }}</h1>
</div>
```

## Notes

- The directive accepts **no value** - providing one will trigger a warning
- It's automatically removed using `queueMicrotask()` after rendering
- Works best when paired with appropriate CSS rules
- Use on root elements rather than multiple child elements for better performance
- The attribute name is `dv-cloak` (no shorthand version)
- Removal happens in a microtask, ensuring content is fully compiled before display
